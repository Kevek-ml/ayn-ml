# ayn-ml Pro

ayn-ml Pro is a commercial extension of [ayn-ml](https://pypi.org/project/ayn-ml/) for teams
that need advanced monitoring capabilities in production. It depends on `ayn-ml` and implements
its public protocols — no Apache 2.0 code is duplicated.

---

## Advanced Advisor

The built-in `MetricAdvisor` generates monitoring plans automatically from your data schema and
task type. ayn-ml Pro unlocks `standard` and `comprehensive` advisor profiles: deeper statistical
analysis, broader drift coverage, and risk-weighted metric selection calibrated to your model type
and data volume.

---

## Cloud Runner

The open-source runner executes monitoring plans in a single synchronous call. Pro's Cloud Runner
adds cron-based scheduling, parallel metric execution, configurable retry logic, and a full audit
trail — built for teams running monitoring inside CI/CD pipelines or production batch jobs.

---

## Slack Notifications

Pro adds a Slack sink: when drift or performance degradation is detected, a formatted monitoring
summary is posted directly to the channel of your choice, with per-metric status and threshold
breach details.

---

## S3 Storage

Pro's `S3Store` persists monitoring history to any S3-compatible object store (AWS S3, MinIO,
GCS via S3 adapter), replacing the default SQLite store for distributed or cloud-native
deployments.

---

## LLM Safety and Quality Metrics

For teams monitoring generative AI outputs, Pro adds:

- **Safety metrics** — toxicity detection and hallucination rate, scored by a model-based classifier
- **LLM-judge quality metrics** — relevance, faithfulness, and coherence, scored by a configurable
  LLM judge (OpenAI or Anthropic)

These complement the deterministic NLP metrics (BLEU, ROUGE, BERTScore, embedding drift) available
in the open-source package.

---

## Agent Metrics

For multi-step agentic workflows, Pro tracks reasoning quality beyond raw outputs: step efficiency
against optimal traces, goal adherence scored by an LLM judge, and cost and latency breakdowns
per tool call.

---

## Explainability

Pro extends the open-source `DriftAttributor` with:

- **SHAP-based drift monitoring** — track feature importance shifts over time alongside distribution drift
- **LLM drift explanation** — natural-language summaries of what drove a drift alert, generated by a judge model
- **Experimental XAI modules** — post-hoc analysis tools for deep-dive investigation of model behaviour changes

---

## Analytics Dashboard

A hosted dashboard for teams managing multiple models and monitoring plans: historical trend
charts, per-feature drill-down, multi-team access control (RBAC), and SSO integration.
