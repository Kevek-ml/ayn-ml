    Sinks¶

    Result sinks fire notifications when alerts are triggered. They implement the ResultSink protocol and receive only the reports that contain fired alerts.

    Protocol¶

    ResultSink ¶

    Bases: Protocol

    Write-only interface for notification channels.

    Implementations must be stateless between calls — each write() is an independent dispatch. Errors raised by write() are caught by the Runner and logged as warnings; they do not abort the run.

    write ¶

    write(report: MonitoringReport) -> None

    Dispatch a monitoring report to the sink destination.

    Any exception raised by this method is caught and logged by the Runner as a warning — it does not abort the run.

    Parameters:

    Name Type Description Default
    report MonitoringReport

    The completed MonitoringReport from a Runner execution.

    		 required

    EmailChannel¶

    EmailChannel ¶

    ResultSink that sends a MonitoringReport summary via SMTP.

    Uses Python's built-in smtplib and email libraries — no third-party mail library is required.

    When renderer is provided, the email is multipart (HTML + plain text fallback). Otherwise only a plain-text body is sent.

    Parameters:

    Name Type Description Default
    host str

    SMTP server hostname (e.g. "smtp.gmail.com").

    		 required
    to_addrs list[str]

    List of recipient email addresses.

    		 required
    from_addr str

    Sender email address. Defaults to the first element of to_addrs when not set.

    		 ''
    port int

    SMTP port. 587 for STARTTLS (default), 465 for SSL, 25 for unencrypted.

    		 587
    username str

    SMTP authentication username. Required by most cloud mail providers.

    		 ''
    password str

    SMTP authentication password or app-specific token.

    		 ''
    subject_template str

    Python format string for the email subject. Available keys: {plan_name}, {model_id}, {model_version}, {n_alerts}.

    		 '[ayn-ml] {plan_name} — {n_alerts} alert(s) fired'
    use_tls bool

    When True (default), use STARTTLS. Set to False for SSL-wrapped connections (port 465) or for local development SMTP servers.

    		 True
    timeout int

    SMTP connection timeout in seconds. Defaults to 30.

    		 30
    renderer HtmlRenderer | None

    Optional HtmlRenderer instance. When supplied, sends a multipart email with an HTML body plus a plain-text fallback.

    		 None

    Example::

    channel = EmailChannel(
    host="smtp.gmail.com",
    to_addrs=["ops@example.com"],
    from_addr="monitoring@example.com",
    username="monitoring@example.com",
    password="app-password",
)

    write ¶

    write(report: MonitoringReport) -> None

    Send the MonitoringReport summary via SMTP.

    Parameters:

    Name Type Description Default
    report MonitoringReport

    Completed MonitoringReport from a Runner execution.

    		 required

    Raises:

    Type Description
    SMTPException

    On SMTP protocol errors.
    OSError

    On network failures.

    WebhookChannel¶

    WebhookChannel ¶

    ResultSink that POSTs a MonitoringReport as JSON to an HTTP endpoint.

    Uses Python's built-in urllib.request — no third-party HTTP library required. The full MonitoringReport.to_dict() payload is sent with Content-Type: application/json.

    Parameters:

    Name Type Description Default
    url str

    Destination URL. Must include the scheme (e.g. "https://hooks.example.com/monitoring").

    		 required
    extra_headers dict[str, str] | None

    Additional HTTP headers to include in the request (e.g. {"Authorization": "Bearer token"}). Merged with the default Content-Type header; caller-supplied values take precedence.

    		 None
    timeout int

    Request timeout in seconds. Defaults to 30.

    		 30

    Example::

    channel = WebhookChannel(
    url="https://hooks.example.com/monitoring",
    extra_headers={"X-API-Key": "secret"},
    timeout=10,
)

    write ¶

    write(report: MonitoringReport) -> None

    POST the MonitoringReport as JSON to the configured URL.

    Serialises the report with MonitoringReport.to_dict(). Raises on HTTP errors (4xx/5xx) and network failures; the Runner catches and logs these.

    Parameters:

    Name Type Description Default
    report MonitoringReport

    Completed MonitoringReport from a Runner execution.

    		 required

    Raises:

    Type Description
    URLError

    On network failures or DNS resolution errors.
    HTTPError

    On HTTP 4xx/5xx responses.
