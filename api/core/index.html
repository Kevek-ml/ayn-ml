    Core¶

    The ayn_ml.core package contains all shared types: plans, schemas, specs, results, reports, and alert primitives.

    Plans¶

    MonitoringPlan ¶

    Bases: BaseModel

    Top-level configuration for a monitoring run.

    A MonitoringPlan binds together the data schema, model identity, the list of metrics to evaluate, and optional data-loading directives. It is the primary input to the Runner.

    Attributes:

    Name Type Description
    name str

    Human-readable name for this plan (used in reports and storage).
    model_id str

    Identifier of the model being monitored.
    model_version str

    Version string of the model (e.g. "1.2.0").
    data_schema DataSchema

    Schema describing the structure of the evaluation DataFrame. Accepts TabularSchema, TextSchema, or AgentSchema.
    metrics list[MetricSpec]

    Ordered list of metric specifications to compute.
    enable_profiling bool

    When True, the Runner computes a statistical profile (min, max, mean, std, percentiles, null rate, etc.) for every feature column referenced by the configured metrics plus the schema target columns (prediction_col, label_col, proba_col) and attaches the result to MonitoringReport.profile. Defaults to False — no profiling overhead on standard runs.
    window WindowConfig | None

    How to narrow the loaded DataFrame to the current monitoring window. None means the source is already pre-filtered.
    sampling RandomSamplingConfig | None

    Optional random subsampling applied after window selection, to reduce the current window size for performance. None means all rows in the window are used.
    description str

    Optional free-text description stored alongside reports.

    Metric specs¶

    MetricSpec ¶

    Bases: BaseModel

    Immutable specification for a single metric computation.

    Attributes:

    Name Type Description
    name str

    Registry name used to look up the metric implementation.
    metric_type MetricType | None

    Category of this metric (see MetricType). Defaults to None; the runner resolves the type from the registry at runtime. Set explicitly only for custom/unregistered metrics.
    feature_name str | None

    Column name for drift and statistics metrics that operate on a single feature. Not required for performance metrics.
    params dict[str, Any]

    Arbitrary keyword arguments forwarded to the metric's compute() method (e.g. {"average": "macro"} for F1, {"q": 0.95} for a quantile).
    threshold float | list[float] | None

    Pass/fail threshold for the computed value. Accepts: - None: no evaluation, MetricResult.status will be None. - float: scalar bound (direction controlled by upper_bound). - list[float]: two-element [lo, hi] range; passes when lo <= value <= hi.
    upper_bound bool

    When True (default), the metric passes when value <= threshold. When False, it passes when value >= threshold. Ignored when threshold is a list.
    metric_type MetricType | None

    Category of this metric. When None (default), the type is resolved at runtime from the metric registry using the name. Provide an explicit value only for custom metrics that are not yet registered.

    MetricType ¶

    Bases: str, Enum

    Enumeration of metric categories supported by ayn-ml.

    Attributes:

    Name Type Description
    performance

    Supervised performance metrics (accuracy, F1, MSE, …).
    drift

    Distribution-shift metrics (PSI, Wasserstein, MMD, …).
    statistics

    Descriptive statistics (mean, std, quantile, …).
    fairness

    Group-level disparity metrics (demographic parity, equalized odds, …).
    nlp_quality

    Text-quality metrics (ROUGE, BLEU, METEOR, …).
    nlp_drift

    Embedding- or token-level distribution shift.
    nlp_safety

    Toxicity, bias, and safety classifiers.
    agent_performance

    Agent task-completion and quality scores.
    agent_cost

    Token usage and cost tracking for agent runs.
    custom

    User-defined metrics registered at runtime.

    Schemas¶

    TabularSchema ¶

    Bases: BaseSchema

    Schema for supervised tabular ML models (classification and regression).

    Attributes:

    Name Type Description
    type Literal['tabular']

    Discriminator literal, always "tabular".
    label_col str

    Column containing ground-truth labels (y_true).
    prediction_col str

    Column containing model predictions (y_pred).
    proba_col str | None

    Column containing predicted probabilities; required for AUC, log-loss, and Brier score. Set to None to disable.
    feature_types dict[str, Literal['numeric', 'categorical']]

    Maps column names to "numeric" or "categorical". Used by drift metrics to determine the appropriate computation path. Columns absent from this dict fall back to numpy dtype inference, which is correct for float/string columns but wrong for integer-encoded categoricals. Use from_dataframe to populate this automatically with optional overrides.
    protected_cols list[str] | None

    Columns representing sensitive or protected attributes (e.g. ["gender", "age_group"]). Fairness metrics read spec.feature_name and require it to be listed here when this field is set. None disables the declaration check (the column must still exist in the DataFrame).

    column_names property ¶

    column_names: list[str]

    Return all non-None column name values declared by this schema.

    from_dataframe classmethod ¶

    from_dataframe(df: Any, feature_types: dict[str, Literal['numeric', 'categorical']] | None = None, **kwargs: Any) -> TabularSchema

    Build a TabularSchema by inferring column types from a DataFrame.

    Only feature columns are inspected — schema columns (label_col, prediction_col, proba_col, timestamp_col, model_id_col, model_version_col) are excluded so they never appear in feature_types and cannot accidentally be fed to drift metrics.

    Remaining columns are classified by numpy dtype:

    • Numeric dtypes (int, float) → "numeric"
    • All other dtypes (object, string, category) → "categorical"

    The optional feature_types argument overrides inferred values for specific columns. Use it to correct integer-encoded categoricals that dtype inference cannot distinguish from genuine numeric features.

    Parameters:

    Name Type Description Default
    df Any

    Source DataFrame (eager pandas or Polars DataFrame). Polars LazyFrame is not supported; call .collect() first.

    		 required
    feature_types dict[str, Literal['numeric', 'categorical']] | None

    Explicit type overrides applied on top of inference. Only columns that would be inferred incorrectly need to be listed here (e.g. {"region": "categorical"}).

    		 None
    **kwargs Any

    Additional TabularSchema fields (label_col, prediction_col, proba_col, etc.).

    		 {}

    Returns:

    Type Description
    TabularSchema

    TabularSchema with feature_types populated from inference
    TabularSchema

    merged with any explicit overrides.

    Results¶

    MetricResult ¶

    Bases: BaseModel

    Computed output for a single metric.

    Context (model_id, timestamp, …) lives on MonitoringReport, not here. MetricResult is a pure computation result: spec in → value + status out.

    Attributes:

    Name Type Description
    spec MetricSpec

    The MetricSpec that drove this computation.
    value float | int | str | None

    Numeric or string result. None when the metric could not produce a value but did not raise (edge-case only).
    status bool | None

    True if the value passes the spec's threshold, False if it fails, or None when no threshold is defined.
    conf_interval tuple[float, float] | None

    Optional (lower, upper) confidence interval for metrics that support bootstrapping (reserved for future use).
    effect_size float | None

    Standardised effect size where applicable (e.g. Cohen's d for t-test, Cliff's delta for Mann-Whitney, variance ratio for Levene, KS D-statistic for ks_2samp). None when the metric does not produce an effect size.
    effect_size_label str | None

    Human-readable identifier for the effect size scale (e.g. "cohen_d", "cliff_delta", "variance_ratio", "ks_statistic"). Always set when effect_size is not None; None otherwise.

    MetricError dataclass ¶

    Structured record of a metric that raised during computation.

    The Runner catches exceptions per-metric and converts them to MetricError rather than aborting the whole run.

    Attributes:

    Name Type Description
    metric_name str

    Registry name of the metric that failed.
    error_type str

    Class name of the exception (e.g. "SchemaError").
    message str

    Human-readable error message.

    MonitoringReport dataclass ¶

    Top-level output of a Runner execution.

    Aggregates all metric results, errors, and fired alerts from a single monitoring run, bound to a plan and execution context.

    Attributes:

    Name Type Description
    plan MonitoringPlan

    The MonitoringPlan that was executed.
    context ExecutionContext

    Identity and time metadata for this run.
    results list[MetricResult]

    Successful MetricResult objects, one per spec.
    errors list[MetricError]

    MetricError objects for specs that failed to compute.
    fired_alerts list[FiredAlert]

    Alerts triggered by AlertRule evaluation (empty when no alert rules are configured).
    profile dict[str, dict[str, float | int | str | None]] | None

    Statistical profile of feature and target columns, populated when plan.enable_profiling=True. None otherwise.

    Execution context¶

    ExecutionContext ¶

    Bases: BaseModel

    Immutable identity and time metadata for a single monitoring execution.

    One ExecutionContext is created per Runner invocation and attached to the resulting MonitoringReport. It is NOT duplicated on individual MetricResult objects — retrieve it from the report instead.

    Attributes:

    Name Type Description
    run_id str

    Unique identifier for this run, auto-generated as a hex UUID. Stable within a run; use it to correlate report entries in a store.
    model_id str

    Identifier of the model being monitored.
    model_version str

    Version string of the model.
    eval_timestamp datetime

    Wall-clock time at which the run was executed (UTC).
    period_start datetime | None

    Start of the data window (min of timestamp_col). None when timestamp_col is not configured in the schema.
    period_end datetime | None

    End of the data window (max of timestamp_col). None when timestamp_col is not configured in the schema.
    n_current int | None

    Row count of the current window after all filtering.
    n_reference int | None

    Row count of the reference window, or None when no reference was provided.

    Alerts¶

    AlertRule dataclass ¶

    Binding between a metric, an alert policy, and notification channels.

    When the Runner evaluates alert_rules, it matches each rule's metric_name to a computed MetricResult and calls policy.evaluate(result). If the policy fires, every channel in channels receives write(report) with the completed MonitoringReport.

    Parameters:

    Name Type Description Default
    metric_name str

    Registry name of the metric to watch (must match a MetricSpec.name in the monitoring plan).

    		 required
    policy AlertPolicy

    Alert policy that decides whether the alert fires.

    		 required
    channels list[ResultSink]

    List of ResultSink implementations to notify when the alert fires. Typically [EmailChannel(...)], [WebhookChannel(...)], or a mix of both.

    		 list()

    Example::

    from ayn_ml.core.alert import AlertRule, ThresholdPolicy
from ayn_ml.sinks.email import EmailChannel

channel = EmailChannel(host="smtp.example.com", to_addrs=["ops@example.com"])
rule = AlertRule(
    metric_name="psi",
    policy=ThresholdPolicy(threshold=0.2),
    channels=[channel],
)
runner.run(plan, df, alert_rules=[rule])

    ThresholdPolicy dataclass ¶

    Stateless threshold-based alert policy.

    Fires when a MetricResult's value crosses the configured threshold. When threshold is None, delegates to the result's pre-computed status field — i.e., fires whenever status is False (meaning the metric's own spec threshold was breached).

    This is the default policy for simple "alert me when the metric is bad" use cases. It requires no historical data and no store access.

    Parameters:

    Name Type Description Default
    threshold float | None

    Custom alert threshold. When None (default), fires when MetricResult.status is False. When set, compares the result's value directly, independently of the spec threshold.

    		 None
    upper_bound bool

    Direction of the custom threshold check. When True (default), fires when value > threshold. When False, fires when value < threshold. Ignored when threshold is None.

    		 True

    Example::

    # Fire when the metric's own threshold is breached
policy = ThresholdPolicy()

# Fire when accuracy drops below 0.80 (regardless of spec)
policy = ThresholdPolicy(threshold=0.80, upper_bound=False)

# Fire when PSI exceeds 0.2
policy = ThresholdPolicy(threshold=0.2, upper_bound=True)

    policy_type: str

    Policy type identifier.

    AlertPolicy ¶

    Bases: Protocol

    Protocol satisfied by all alert policies.

    An AlertPolicy is a stateless callable object: given a single MetricResult it returns True when the alert should fire.

    Implementations must also expose:

    • policy_type — short string identifier (e.g. "threshold").
    • details(result) — dict of policy-specific context attached to the FiredAlert record.

    The Runner calls evaluate() once per (AlertRule, MetricResult) pair.

    policy_type property ¶

    policy_type: str

    Short string identifying the policy kind (e.g. "threshold").

    evaluate ¶

    evaluate(result: MetricResult) -> bool

    Return True when the alert condition is met for result.

    details ¶

    details(result: MetricResult) -> dict[str, Any]

    Return policy-specific context for the FiredAlert record.

    FiredAlert dataclass ¶

    Lightweight record of an alert triggered during a monitoring run.

    FiredAlert is intentionally minimal — it carries only what is needed to log or notify. Full AlertRule configuration lives in the alert layer.

    Attributes:

    Name Type Description
    metric_name str

    Registry name of the metric that triggered the alert.
    policy_type str

    Policy kind: "threshold", "change", or "consecutive".
    details dict[str, Any]

    Policy-specific context, e.g. {"threshold": 0.8, "value": 0.7} for a threshold alert.
    feature_name str | None

    Feature column the result belongs to, or None for global (plan-level) metrics such as performance and target drift. Populated when a single alert rule matches multiple per-feature results (e.g. psi/age and psi/income).

    Exceptions¶

    AynError ¶

    Bases: Exception

    Base exception for all ayn-ml errors.

    Catch this class to handle any library-specific error uniformly.

    SchemaError ¶

    Bases: AynError

    Raised when a required column is missing or the schema is incompatible.

    Examples:

    Raised when TabularSchema.label_col does not exist in the DataFrame, or when a drift metric is given a TextSchema.

    MetricComputeError ¶

    Bases: AynError

    Raised inside compute() for unexpected errors not covered by other types.

    The Runner catches this per-metric and records it as a MetricError rather than aborting the full monitoring run.

    UnknownMetricError ¶

    Bases: AynError

    Raised when a metric name is not found in the registry.

    Attributes:

    Name Type Description
    message

    Human-readable description including the list of known metrics.

    InsufficientDataError ¶

    Bases: AynError

    Raised when the DataFrame has too few rows to compute a metric reliably.

    Examples:

    Raised when fewer than 2 rows are passed to a performance metric, or when all y_true values are zero for MAPE.

    ThresholdError ¶

    Bases: AynError

    Raised when a MetricSpec contains an invalid threshold configuration.

    Examples:

    Raised when a list threshold has fewer than two elements.
